VirtuOps

Group 16's Capstone Project

By: Zeerak.A | Jacob.H | Deven.V | Brant.L

Introduction

VirtuOps is a Virtual Machine Management app that was made for ELEC 377, an operating system course. Before this app was implemented teams of students were designated virtual machines to undergo various lab tasks. Each year, these machines were manually created and managed by Dr. Dean and the IT team. Hypothetically, if an issue appeared, the students are not able to fix their own machines. A recent example of this occurred this term, where students would fill up their disk drive and be locked out of their machines, requiring manual intervention by the professor. To resolve such an issue and create more ease for Dr. Dean, the graduate teaching assistants, and the IT team, our team decided to implement a lightweight, singlenode VM management system that would simplify troubleshooting. VirtuOps is designed to be a system with role-based access control such that students can sign-in and manage their VM.

Software Design

Users

VirtuOps's target audience is the following types of users:

  • Undergraduate Students
  • Graduate Research Students
  • Professor
  • Undergraduate & Graduate Teaching Assistants

Main Functionalities

Core features include Start, Stop, Snapshot VMs. Additional features and custom capabilities catered for the Professor, Research Students, TAs, and Admins include creating, deleting, and listing VMs.

Backend

The backend consists of:

  • Nginx: The web server.
  • SQLite: The database that holds all user information.
  • Django: For the authentication.
  • Automated scripts for all the main functionalities.
  • Custom OS with metadata, user information, and network configuration.

Analysis

Analyzed and conducted meetings with the professor and the IT team to review the software stack, library, authentication, and general advice to ensure a more efficient solution is reached. The team graded a few design choices using a criterion table. This criterion table encourages constant inspection throughout each life cycle stage to ensure the software is easily accessible, safe, and efficient.

Challenges

  • Scalability: Ensuring the system can efficiently handle an increasing number of users and VM instances without any performance degradation.
  • Security: Prioritizing safety to ensure no unauthorized access or SQL injection occurs, and to maintain data integrity by respecting data privacy for all users' virtual machines.
  • User Usability: Ensuring the software is user-friendly and accommodates users with varying levels of technical expertise.
  • Resource Allocation: Managing the physical resources (CPU, memory, storage) of the host system to prevent overcommitment and ensure optimal performance for all VMs.
  • Fault Tolerance and Recovery: Implementing robust mechanisms to recover from hardware or software failures with minimal disruption to users.
  • Network Configuration and Performance: Ensuring reliable and efficient network connectivity for VMs, including handling virtual network configurations and potential bottlenecks.
  • Integration with Existing Infrastructure: Seamlessly integrating with the existing IT infrastructure, including authentication systems, storage solutions, and network setups.

Recommendations

  • Prioritizing User Experience: The interface is intuitive and easily accessible.
  • Emphasis on Security: To implement strong authentication and encryption to protect user data and VMs.
  • Test Thoroughly: Constant testing should be conducted, which also includes stress tests, to spot and fix issues early.
  • User Feedback: Regularly gather feedback to improve the software and add new features.

Next Steps

  • Enhance Security Protocols: Continuously update and strengthen security measures to address emerging threats and vulnerabilities.
  • Cloud Integration: Integrate cloud services to increase scalability.
  • Feedback Feature: Implementing a feedback feature that enables the user to provide feedback on their likes and dislikes of the app.
